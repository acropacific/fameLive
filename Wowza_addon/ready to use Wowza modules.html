  • Module Collection

    Published on 10-02-2010 05:40 AM      Number of Views: 51324  
    This article describes a collection of utility modules that are ready to use in Wowza® media server applications. (The Eclipse IDE is NOT required.)

    Notes:
    • Modules in the Wowza Streaming Engine Module Collection are designed for use with Wowza Streaming Engine™ software version 4.0 and later. Most of the modules are backward-compatible; however, some of the modules aren't fully supported in Wowza Media Server® software. When a module requires a specific minimum version of the Wowza media server software to function, it will be noted in the module article.

    • The Module Collection download package includes source code that you can use to see how the modules are implemented and to extend the modules for your own use.

    Wowza Streaming Engine Module Collection

    Note: The current Module Collection version is build 12931. See the version.txt file in the Module Collection package for details of changes.

    To install the module collection for Wowza Streaming Engine

    1. Download the Module Collection package: WowzaServerAddOnCollection_4.0.zip.

    2. Extract the contents from the downloaded (zipped) package and then copy /lib/wms-plugin-collection.jar from the package to the /lib folder in your Wowza Streaming Engine installation ([install-dir]/lib.

    3. To configure a specific module in the collection, refer to the articles in the Modules List below.

    4. Restart Wowza Streaming Engine.

    Modules List

    Module Name Module Description
    MediaCacheLocalFirstRandomAccessReader This custom IRandomAccessReader implementation looks for content in an application's content storage directory first and, if it can't be found, tries to retrieve the content from a remote Media Cache source. This enables you to make locally stored content available before it's transferred to the remote server.
    MediaCachePreload This server listener enables you to specify files to preload into the Media Cache before first playback. An example would be a known popular video that you want to initially set in Media Cache so that you can avoid any "first-read" delays that can result from the inability to read from the source quickly enough.
    MediaCacheSourceSelector When using Synchronized Multimedia Integration Language (SMIL) files to define adaptive bitrate (ABR) streams, the VOD assets are normally required to be stored in the same folder as the SMIL file that references them. This custom RandomAccessReader class enables a SMIL file and it's associated VOD assets to be in different Media Cache Sources.
    ModuleAddAudioTrack Adds an audio track to a live stream that doesn't have audio.
    ModuleClientBWCheck Used with a Flash client to test the bandwidth available from the Flash client to the server (client-to-server bandwidth).
    ModuleCreateSnapshot Takes a single-frame snapshot of a live stream for thumbnail creation.
    ModuleCupertinoLiveAttachPicture Illustrates how to add poster images to an iOS audio-only live stream that is produced by using the audio-only rendition of the App Store compliance technique.
    ModuleCupertinoLiveAttachPicture2
    ModuleCupertinoVODAttachPicture    		 Illustrates how to add poster images to an iOS audio-only live or video on demand stream.
    ModuleDuplicateStreams Copies a stream from one application instance to another on the same Wowza media server.
    ModuleDvrRecorderControl Controls which streams get recorded by the Wowza nDVR AddOn recorder. It can be used with Wowza Transcoder AddOn or any time where you don't want all streams to be recorded.
    ModuleFixStreamPath Fixes the stream path where some Flash players that accept a single string for RTMP connection and stream name don't separate them correctly when content is stored in a subfolder of an application's content location.
    ModuleGetConnectionCount Gets connection counts for server, applications, and application instances sent to a Flash application.
    ModuleHotlinkDenial Sets a list of website domain names whose HTML pages are allowed to embed the Flash clients that connect to your application.
    See Also: ModuleRefererValidate
    ModuleLimitPublishedStreamBandwidth Monitors bandwidth of published streams.
    ModuleLimitStreamViewers Limits the number of viewers to a stream.
    ModuleLoadBalancerEdgeLimitConnections Stops load balancer redirects when the edge server limit has been reached.
    ModuleLogViewerCounts Logs client count and metadata for a stream every 10 seconds.
    ModuleLoopUntilLive Loops a server-side filler video stream in place of a live stream that isn't available.
    ModuleMultiTrackVOD Selects which tracks in a video on demand (VOD) file that has multiple video, audio, or data tracks, to a player. The module uses query parameters that are added to the stream URLs to tell the server which tracks to select.
    ModuleOnConnectAuthenticate Performs file-based authentication with RTMP clients with credentials passed as parameters in the NetConnection.connect method.
    ModuleOnConnectAuthenticate2 Performs file-based authentication with RTMP clients with credentials passed as query parameters in the RTMP URL.
    ModulePacketizerControl Controls which streams get packetized for HTTP streaming. It can be used with Wowza Transcoder AddOn or any time where you don't want all streams to be packetized.
    ModuleRefererValidate Determines the identity of the webpage that a player is launched from using an image tag or JavaScript embedded in the webpage. It's a simple but effective alternative to the ModuleHotlinkDenial module.
    ModuleStreamControl Controls Stream class streams and playlists. It can be used to dynamically create, set up, and start new live Stream class streams from static and live sources and add sources to existing Stream class streams. It uses a custom Flash client as a user interface to the module.
    ModuleStreamWatchDog Monitors live streams that are published to an application and broadcasts information to connected Flash clients every 10 seconds. The Flash clients can be configured to respond to these broadcasts, either to monitor the stream health or to playback the stream.
    ModuleTimedDisconnect Disconnects players from an application after an elapsed time period.
    ModuleTranscoderControl Controls which streams get transcoded with Wowza Transcoder AddOn. It can be used when you don't want all streams that are published to the server to be transcoded.
    ModuleTranscoderTimedSnapshot Takes a thumbnail snapshot of a Wowza Transcoder stream at regular intervals. You can save a sequence of time-stamped images or a single image that's overwritten continuously.
    ModuleVideoNameList Gets a list of files in your application's content location.
    ServerListenerLoadAndLockAppInstances Used to load an application when Wowza Streaming Engine starts and remains loaded until the server is shut down. The server listener can be configured to load multiple applications or application instances on the server.
    ServerListenerStreamPublisher Performs scheduled streaming of streams and playlists in Wowza Streaming Engine.
    ServerListenerUDPPortRecycleTime Enables you to change the amount of time required to re-allocate released UDP ports to new IP camera connections.

    Changes from the Previous Module Collection

    The modules in the Removed Modules List below have been removed from the Wowza Streaming Engine Module Collection. They have either been replaced by core functionality in the Wowza Streaming Engine software or are no longer required. The module configuration articles have been retained in the list as a reference.

    If you still need to use one of these modules, either because you're still using Wowza Media Server software or your workflow doesn't allow you to use the core features in Wowza Streaming Engine, you can download the following package that contains the removed modules: Download WowzaServerAddOnCollection.zip.

    Removed Modules List

    Module Name Module Description
    ModuleBlockDuplicateStreamNames Prevents a live encoder from publishing a stream that already exists in the same application instance.
    ModuleCalculateBitrate Calculates the bitrate of a video on demand (VOD) stream.
    ModuleEncryptionHandlerCupertinoStreaming Uses the internal method of AES-128 encryption to secure a live or VOD stream to send to Apple iOS devices.
    ModuleFCSubscribe Implements a server-side method for sending FCSubscribe messages to a Content Delivery Network (CDN).
    ModuleLimitConnectionsToApplication Limits the number of connections to an application.
    ModuleOverridePlayBlackListIP Overrides play to blacklist (refuse connection) to certain IP addresses.
    ModuleOverridePlayRestrictIP Overrides play to limit Flash clients to certain IP addresses. Handling for HTTP and RTSP clients is also included.
    ModuleOverridePublishRestrictIP Limits publishing of live streams to allowed IP addresses.
    ModuleRequireSecureConnection Enables a requirement for RTMPE, RTMPTE, or RTMPS to be used to connect to a given Wowza Media Server application.
